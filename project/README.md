## 4. Как запустить проект

Опишите, как именно воспроизвести основные сценарии использования проекта.

Примеры (заполните под свой случай):

### 4.1. Запуск обучения модели

```bash
cd project
source .venv/bin/activate

# Полный цикл: загрузка данных → обучение → сохранение артефактов
python -m src.models.train

# Или с явными путями
python -m src.models.train --data data/creditcard.csv --out models/
```

Артефакты сохраняются в `models/`: `final_model.pkl`, `preprocessor.pkl`, `threshold.json`, `model_card.md`.

### 4.2. Запуск сервиса (API)

```bash
cd project
source .venv/bin/activate
uvicorn src.service.main:app --host 0.0.0.0 --port 8000
```

Или через Docker:

```bash
cd project
cp configs/.env.example .env   # заполнить реальными значениями
docker compose up --build
```

Сервис поднимается на `http://localhost:8000`. Ключевые эндпоинты:
- `POST /predict` — предсказание fraud/legit по транзакции
- `GET /health` — статус сервиса и версия модели
- `GET /metrics` — счётчики запросов и latency (Prometheus-формат)
- `GET /docs` — Swagger UI для ручного тестирования

## 8. Мониторинг и наблюдаемость

Сервис экспортирует метрики в Prometheus-формате на `GET /metrics`:

| Метрика | Тип | Описание |
|---------|-----|----------|
| `fraud_api_requests_total` | Counter | Количество запросов по эндпоинту и статусу |
| `fraud_api_request_duration_seconds` | Histogram | Latency запросов (p50, p95, p99) |
| `fraud_predictions_total` | Counter | Количество предсказаний fraud/legit |
| `fraud_model_threshold` | Gauge | Текущий порог классификации |

Структурированные логи пишутся в stdout в формате:
```
2024-01-15 12:34:56 INFO fraud_api predict amount=149.62 prob=0.0023 result=legit
```

Для просмотра в реальном времени через Docker:
```bash
docker compose logs -f fraud-api
```

## 9. Ограничения и дальнейшая работа

**Текущие ограничения:**
- Признаки V1–V28 анонимизированы через PCA — feature engineering невозможен
- Датасет 2013 года, паттерны мошенничества могли измениться
- Нет мониторинга data drift в продакшне

**Направления для развития:**
- Добавить мониторинг дрейфа данных (Evidently)
- Реализовать explainability через SHAP в ответе API
- Добавить онлайн-дообучение модели на новых транзакциях

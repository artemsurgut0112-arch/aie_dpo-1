# HW14 – эмбеддинги, FAISS, оценка retrieval и mini-RAG по базе знаний

## 1. Кратко: что сделано

- Использовалась база знаний из 10 документов о основах Python программирования.
- Чанкинг документов выполнялся с chunk_size=50 и overlap=20.
- Использовалась embedding-модель 'all-MiniLM-L6-v2' от sentence-transformers.
- Базовая проверка retrieval показала hit@3=0.8.
- После обновления базы знаний retrieval улучшился для новых тем.
- Mini-RAG собирал контекст из top-3 чанков и генерировал простой ответ.

## 2. Среда и воспроизводимость

- Python: 3.8+
- faiss: 1.7+
- sentence-transformers: 2.2+
- torch: 1.9+
- Устройство: CPU
- Seed: 42
- Как запустить: открыть `HW14.ipynb` и выполнить Run All.

## 3. База знаний и подготовка документов

- Предметная область: Основы программирования на Python.
- Источник документов: Синтетические документы, созданные для задания.
- Число исходных документов: 10
- Число чанков после разбиения: 50
- Какой способ чанкинга использовался: Фиксированный размер с перекрытием.
- Основные параметры чанкинга: chunk_size=100, overlap=20
- Комментарий: База знаний охватывает базовые концепции Python, что подходит для retrieval-задач, так как запросы могут быть о конкретных конструкциях языка.

## 4. Retrieval и индекс

### 4.1. Эмбеддинги и индекс

- Какая embedding-модель или способ векторизации использовались: sentence-transformers 'all-MiniLM-L6-v2'
- Какая метрика сходства использовалась: Косинусное сходство (inner product после нормализации)
- Какой индекс `FAISS` использовался: IndexFlatIP
- Какой `top_k` использовался по умолчанию: 3
- Что показали первые примеры retrieval: Для запроса "Что такое переменные в Python?" найдены релевантные чанки с высокой похожестью.

### 4.2. Контрольные запросы и оценка retrieval

- Сколько контрольных запросов использовалось: 10
- Что считалось релевантным ответом: Совпадение с ожидаемым документом по индексу.
- Какие метрики retrieval считались: hit@3, recall@3
- Итоговый `hit@k`: 0.8
- Итоговый `recall@k`: 0.8
- Дополнительные метрики: rank_of_first_relevant
- Какие запросы оказались самыми простыми: Запросы с точными терминами, как "Что такое список?"
- Какие запросы оказались самыми проблемными: Общие запросы, где несколько документов релевантны.

## 5. Эксперимент по параметрам retrieval и обновление базы знаний

- Какой параметр сравнивался: chunk_size
- Какие два варианта сравнивались: 50 и 100
- Как изменилось качество retrieval: При chunk_size=100 hit@3=0.7, при 50=0.8
- Какой вариант вы выбрали как основной и почему: chunk_size=50, так как дает лучшее качество retrieval.
- Что именно было обновлено в базе знаний: Добавлено 3 новых документа о условных операторах, строках и файлах.
- Как была выполнена переиндексация: Пересозданы чанки, эмбеддинги и индекс FAISS.
- Что изменилось в retrieval после обновления базы знаний: Для новых тем retrieval нашел релевантные чанки, для старых - без изменений.

## 7. Анализ

База знаний оказалась подходящей для учебной задачи, так как документы короткие и тематические. Чанкинг с overlap помог сохранить контекст. Формальная оценка retrieval полезна для понимания качества. Эксперимент показал, что больший chunk_size лучше. Обновление базы знаний расширило coverage. Ошибки mini-RAG связаны с простым генератором ответов, без настоящей LLM. Retrieval играет ключевую роль в качестве финального ответа. Ограничения: отсутствие разнообразных embedding-моделей и настоящего генератора.

## 8. Итоговый вывод

Базовый retrieval-конвейер с FAISS и sentence-transformers подходит для такой задачи. Главное понятие - качество retrieval напрямую влияет на RAG. Для production нужны более сложные компоненты.
